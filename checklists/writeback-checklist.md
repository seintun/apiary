# Writeback Checklist

Write back only if the result is:

- durable,
- reusable,
- decision-worthy,
- safety-relevant,
- likely to prevent future mistakes.

Choose one destination:

- decision record
- runbook/playbook
- project docs
- issue/PR comment
- changelog
- team wiki
- personal/agent memory

Do not write back raw worker output unless audit requires it.
